## 🛠️ Phase 1: Product Development (Creating the Assets)

### 1. The "Master Rules" Library (`.cursorrules`)
We will create highly specific, opinionated rule files for the most popular stacks.

**Variants to create:**
- [ ] **The "Next.js Supreme" Rule**: Enforces Next.js App Router, shadcn/ui integration, TypeScript strict mode, Tailwind v4 best practices.
- [ ] **The "React Native Mobile" Rule**: Enforces Expo Router, React Native Paper/Tamagui, specialized styling.
- [ ] **The "Python Backend" Rule**: FastAPI/Django specific, Type hinting, Pydantic, Docstring standards.
- [ ] **The "Golang Microservice" Rule**: Project layout standards, error handling patterns.
- [ ] **The "General Typescript" Rule**: Strict typing, no `any`, modern syntax only.

### 2. The "Prompt Engineering" Vault (Markdown Files)
A collection of copy-paste prompts designed for predictable output.

**Categories:**
- [ ] **Refactoring**: "Refactor this component to use Composition Pattern without changing external behavior."
- [ ] **Testing**: "Generate comprehensive Jest unit tests covering edge cases for this function."
- [ ] **Documentation**: "Write JSDoc comments for this file emphasizing parameter constraints."
- [ ] **Debugging**: "Analyze this error stack trace and suggest 3 potential root causes."

### 3. VS Code / Cursor Optimized Profiles (`settings.json`)
A `settings.json` snippet optimized for AI interaction.
- [ ] Improved font ligatures (Fira Code/JetBrains Mono).
- [ ] Minimal UI to focus on code + chat.
- [ ] Keybindings for fast AI chat toggling.
